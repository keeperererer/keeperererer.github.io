为什么前端监控要用GIF打点？

2019-08-03

为什么通常在发送数据埋点请求的时候使用的是 1x1 像素的透明 gif 图片？

  • 能够完成整个 HTTP 请求+响应（尽管不需要响应内容）
  • 触发 GET请求之后不需要获取和处理数据、服务器也不需要发送数据
  • 跨域友好
  • 执行过程无阻塞
  • 相比 XMLHttpRequest 对象发送 GET 请求，性能上更好
  • GIF的最低合法体积最小（最小的BMP文件需要74个字节，PNG需要67个字节，而合法的GIF，只需要43个字节）
